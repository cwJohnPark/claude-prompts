# Freemium 비즈니스 모델 설계 가이드

## 개요

앱의 Freemium 모델을 설계하고 최적화하기 위한 전략 가이드. iOS, Android, watchOS 앱에 적용 가능.

---

## Freemium 모델 유형

| 모델 | 설명 | 적합한 앱 |
|------|------|----------|
| **기능 제한형** | 핵심 기능 잠금 | 생산성, 유틸리티 |
| **용량 제한형** | 저장공간/횟수 제한 | 클라우드, 백업 |
| **시간 제한형** | 체험 기간 후 결제 | 전문 도구, B2B |
| **광고 제거형** | 광고 있음/없음 | 게임, 미디어 (watchOS 불가) |
| **하이브리드** | 위 모델 조합 | 대부분의 앱 |

---

## 핵심 설계 원칙

### 1. 가치 인식 우선 (Value First)

```
잘못된 접근: 핵심 기능을 처음부터 잠금
올바른 접근: 가치를 충분히 경험 -> 더 원하게 만들기
```

**실행 전략:**
- 무료 버전에서도 "아하 모먼트" 경험 가능하게 설계
- 프리미엄 기능 미리보기 제공 (일부 무료 체험)
- 제한에 도달할 때 자연스럽게 업그레이드 제안

### 2. 명확한 가치 차별화

무료/유료 경계가 명확해야 전환 동기 부여.

```
좋은 예시 (피트니스 앱):
├── 무료: 기본 운동 10개, 진행 추적
└── 유료: 200+ 운동, AI 코칭, 맞춤 플랜, 오프라인 모드

나쁜 예시:
├── 무료: 거의 모든 기능
└── 유료: 테마 색상 변경 (가치 불명확)
```

### 3. 적절한 마찰 설계

- **너무 적은 마찰**: 전환 동기 없음
- **너무 많은 마찰**: 사용자 이탈

**균형점**: 무료 사용자도 만족하지만 "조금 더" 원하게. 제한이 "짜증"이 아닌 "아쉬움"으로 느껴지도록.

## 기능 분리 프레임워크

### 가치 매트릭스

```
              높은 가치
                 │
    ┌────────────┼────────────┐
    │   무료     │   유료     │
    │  (훅 기능)  │  (핵심)    │
────┼────────────┼────────────┼──── 사용 빈도
    │   무료     │   유료     │
    │  (기본)    │  (프리미엄) │
    └────────────┼────────────┘
                 │
              낮은 가치
```

**사분면별 전략:**
- **고가치 + 고빈도**: 핵심 유료 기능 (전환 드라이버)
- **고가치 + 저빈도**: 프리미엄 부가 기능
- **저가치 + 고빈도**: 무료 (사용자 유지용)
- **저가치 + 저빈도**: 무료 기본 기능

### 기능 분류 체크리스트

**무료로 제공할 기능:**
- [ ] 앱의 핵심 가치를 경험할 수 있는 기능
- [ ] 일상적으로 자주 사용하는 기본 기능
- [ ] 경쟁사 무료 버전과 동등하거나 우수한 기능
- [ ] 바이럴/공유를 유발하는 기능

**유료로 전환할 기능:**
- [ ] 파워 유저에게만 필요한 고급 기능
- [ ] 시간 절약/효율 향상 기능
- [ ] 개인화/커스터마이징 기능
- [ ] 무제한 사용 (제한 해제)
- [ ] 프리미엄 콘텐츠/데이터

## 가격 책정 전략

### 앱 유형별 권장 가격대 (한국 기준)

| 앱 유형 | 일회성 구매 | 월 구독 | 연 구독 |
|---------|------------|--------|--------|
| 유틸리티 (단순) | ₩1,100~₩4,900 | - | - |
| 유틸리티 (복잡) | ₩4,900~₩14,000 | ₩1,100~₩2,500 | ₩9,900~₩19,000 |
| 생산성 | ₩9,900~₩29,000 | ₩2,500~₩5,900 | ₩19,000~₩49,000 |
| 피트니스/건강 | - | ₩4,900~₩9,900 | ₩39,000~₩79,000 |
| 크리에이티브 | ₩14,000~₩49,000 | ₩5,900~₩14,000 | ₩49,000~₩99,000 |
| watchOS 전용 | ₩1,100~₩4,900 | ₩1,100~₩2,500 | - |

### 가격 심리학

**앵커링 효과:**
```
연간 구독: ₩59,000 (월 ₩4,917)
월간 구독: ₩7,900 <- 연간이 38% 저렴해 보임
```

**가격 끝자리:**
- ₩X,900: 가장 일반적, 저렴해 보임
- ₩X,000: 프리미엄 느낌

## 전환율 최적화

### 전환 유도 최적 타이밍

**좋은 시점:**
1. **성취 직후**: 목표 달성 축하 + 프리미엄 제안
2. **제한 도달**: 자연스러운 업그레이드 프롬프트
3. **습관 형성 후**: 7-14일 사용 후 제안
4. **특별 이벤트**: 할인, 무료 체험 기간

**피해야 할 시점:**
- 앱 첫 실행 직후 (가치 미경험)
- 작업 중간에 갑작스러운 방해
- 사용자 좌절 상황

### 전환 UI 패턴 예시

```swift
struct UpgradePromptView: View {
    let onUpgrade: () -> Void
    let onDismiss: () -> Void

    var body: some View {
        VStack(spacing: 16) {
            // 달성한 것 강조
            Text("10개 프로젝트 완료!")
                .font(.headline)

            // 가치 제안
            Text("프리미엄으로 무제한 프로젝트와\n고급 분석 기능을 사용해보세요")
                .multilineTextAlignment(.center)
                .foregroundColor(.secondary)

            // 명확한 CTA
            Button("프리미엄 시작하기", action: onUpgrade)
                .buttonStyle(.borderedProminent)

            // 부담 없는 탈출구
            Button("나중에", action: onDismiss)
                .foregroundColor(.secondary)
        }
        .padding()
    }
}
```

### 핵심 지표 (KPIs)

| 지표 | 설명 | 벤치마크 |
|------|------|----------|
| **전환율** | 무료->유료 전환 비율 | 2-5% (일반), 10%+ (우수) |
| **ARPU** | 사용자당 평균 매출 | 앱 유형별 상이 |
| **LTV** | 고객 생애 가치 | CAC의 3배 이상 |
| **이탈률** | 구독 취소율 | 월 5-7% (양호) |

## 사용자 심리와 전환

### 전환을 유도하는 심리적 원리

#### 1. 손실 회피 (Loss Aversion)

```
잘못된 표현: "프리미엄으로 업그레이드하세요"
효과적 표현: "무료 체험이 3일 후 종료됩니다"
효과적 표현: "저장된 데이터를 계속 유지하려면..."
```

#### 2. 호혜성 (Reciprocity)

무료로 가치를 먼저 제공하면 보답하고 싶은 심리 발생.
- 충분한 무료 가치 제공 (쪼잔하지 않게)
- 프리미엄 기능 일부 무료 체험 허용

#### 3. 사회적 증거 (Social Proof)

```swift
Text("100,000명 이상의 사용자가 프리미엄을 선택했습니다")
Text("4.8 평점 (15,000개 리뷰)")
```

#### 4. 희소성 (Scarcity)

```
실제 한정 할인: "신년 맞이 50% 할인 (1월 31일까지)"
가짜 희소성은 피하기: 항상 "마지막 기회!" (신뢰 상실)
```

### 전환 장벽과 해결책

| 장벽 | 심리 | 해결책 |
|------|------|--------|
| 가격 | "돈이 아깝다" | 일일 비용 환산, 무료 체험, 환불 보장 |
| 불확실성 | "내가 실제로 쓸까?" | 무료 체험, 미리보기, 사용 시나리오 |
| 결정 피로 | "옵션이 너무 많다" | 플랜 수 최소화, "추천" 표시 |

## 앱 유형별 Freemium 모델 예시

### 생산성 앱 (할일/노트)

```
무료:
├── 기본 메모/할일 작성
├── 3개 폴더 제한
└── 기기 간 동기화

프리미엄 (₩4,900/월 또는 ₩39,000/년):
├── 무제한 폴더
├── 태그, 필터, 고급 검색
├── 위젯
└── 암호 잠금
```

### 피트니스 앱

```
무료:
├── 기본 운동 추적
├── 7일 히스토리
└── 커뮤니티 접근

프리미엄 (₩7,900/월 또는 ₩59,000/년):
├── 무제한 히스토리
├── 상세 분석/리포트
├── AI 운동 추천
└── 오프라인 모드
```

### watchOS 전용 앱 (미니 유틸리티)

```
무료:
├── 핵심 기능 1개 (예: 타이머)
└── 기본 컴플리케이션

프리미엄 (₩2,500 일회성):
├── 추가 기능 (스톱워치, 알람)
├── 모든 컴플리케이션
└── 커스텀 테마
```

## 피해야 할 안티패턴

### 강압적 업그레이드
```
전면 팝업으로 무료 사용 방해
"업그레이드하지 않으면 계속 사용 불가"
닫기 버튼 숨기기/작게 만들기
```

### 기만적 표현
```
가짜 카운트다운 ("지금만 50% 할인!")
가짜 리뷰/통계
숨겨진 비용
```

### 과도한 제한
```
무료 버전을 사실상 사용 불가능하게 만들기
갑작스러운 기능 축소
"n회 사용" 후 완전 잠금
```

## 신뢰 구축 패턴

```
투명한 가격 정책
쉬운 구독 취소
무료 사용자도 존중하는 태도
업그레이드 없이도 핵심 가치 제공
```
